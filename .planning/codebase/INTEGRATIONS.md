# External Integrations

**Analysis Date:** 2026-01-31

## APIs & External Services

**AI/LLM:**
- Claude (Anthropic) - Natural language project command processing
  - SDK: @anthropic-ai/sdk 0.24.0
  - Auth: Claude API key (`claudeApiKey`)
  - Implementation: `src/services/claude.ts`
  - Model used: `claude-3-haiku-20240307`
  - Browser mode: Enabled via `dangerouslyAllowBrowser: true` in `src/services/claude.ts` line 53

## Data Storage

**Databases:**
- None - This is a desktop app with local-only storage

**File Storage:**
- Local filesystem (macOS)
  - Projects data: Electron userData path → `projects.json`
  - Path: Generated by `app.getPath('userData')` in `electron/main.ts` line 17
  - Format: JSON with projects array and settings object
  - Persistence: IPC-based save via `electron/main.ts` lines 78-88
  - Recovery: Load on app startup via `electron/main.ts` lines 66-76

**Caching:**
- None - No external caching configured

## Authentication & Identity

**Auth Provider:**
- None - Manual API key entry
  - Users manually enter Anthropic Claude API key in Settings UI
  - Stored locally in `projects.json` under `settings.claudeApiKey`
  - Retrieved at runtime and passed to Anthropic SDK
  - Implementation in `src/components/Settings.tsx` lines 14-20

## Monitoring & Observability

**Error Tracking:**
- None - No external error tracking service

**Logs:**
- Console logging only
- Development: `mainWindow.webContents.openDevTools()` in `electron/main.ts` line 45
- Error handling in `src/services/claude.ts` lines 111-118 and 162-163
- Errors logged to console, returned to user in UI

## CI/CD & Deployment

**Hosting:**
- Desktop application (no cloud hosting)
- Packaged as macOS DMG and ZIP via electron-builder

**CI Pipeline:**
- None configured
- Manual build: `npm run build`
- Manual packaging: `npm run electron:build` (macOS only)

## Environment Configuration

**Required env vars:**
- None - All critical configuration done via Settings UI
- `VITE_DEV_SERVER_URL` - Set by vite-plugin-electron during development (internal)

**Secrets location:**
- Stored in local `projects.json` file (Electron userData directory)
- No .env file used
- API key stored in plaintext in local storage (user responsibility to protect)

## File System Access

**Reading:**
- Project directory scanning: `electron/scanner.ts`
  - README file scanning
  - package.json parsing
  - Directory listing for project detection

**Writing:**
- Projects data persistence: `electron/main.ts`
  - Path: `app.getPath('userData')/projects.json`
  - Created on first save, overwritten on subsequent saves

**IPC Channels (Electron Bridge):**

**Main → Renderer:**
- `focus-project` - Tray menu focus project event
- `trigger-scan` - Tray menu project scan trigger

**Renderer → Main:**
- `load-projects` - Retrieve saved projects data
- `save-projects` - Persist projects to disk
- `scan-projects` - Scan filesystem for projects
- `read-project-files` - Read README and package.json from project
- `show-window` - Bring app window to focus
- `get-theme` - Get system dark/light theme preference
- `select-folder` - Open native folder picker dialog

## Webhooks & Callbacks

**Incoming:**
- None

**Outgoing:**
- None - Unidirectional API calls to Claude only
